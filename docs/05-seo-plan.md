# SEO Plan (MVP)

## Positioning

- Primary audience: freelancers handling difficult client communication with money, scope, and negotiation pressure.
- Current phase priority: `payment communication` first.
- Keyword intent: high-intent, problem-first, action-oriented.

## Phase Order

1. `payment communication`
2. `scope / boundary communication`
3. `discount / negotiation communication`

Do not expand all clusters at once.
Build one complete cluster before widening coverage.

## Current Phase-1 System

Use the payment cluster as the first structured SEO -> product handoff system:

- `/payment` = hub
- `/guides/*` = guide
- `/templates/*` = template
- `/scenario/*` = scenario

Execution contract:

- `product/seo/payment-cluster-phase1-execution-spec.md`
- `product/seo/payment-cluster-phase1.manifest.json`
- `product/seo/payment-cluster-phase1.codex-prompt.md`

## Guide Content Contract

Every guide should include:

1. Problem framing
2. Why this is hard / common mistake
3. Decision or communication principles
4. Situation branches
5. Example wording
6. What to do next
7. Natural transition to template or scenario

## Structural SEO Rules

- Build `Hub -> Guide -> Template -> Scenario`, not isolated pages.
- Keep high-density internal links within a cluster.
- Keep low-density links across clusters.
- Do not publish thin template reskins, thin tool shells, or keyword-permutation pages.

## Indexing Gate (Required)

- Dev/preview: `noindex,nofollow` + `robots disallow`.
- Launch-ready only: open indexing + submit sitemap.

## Launch Criteria for Indexing On

- Output quality stable across core use cases.
- No thin or duplicate autogenerated pages.
- The payment cluster reads as one coherent topical network.
- At least one useful, non-generic answer exists for each owned phase-1 query group.
