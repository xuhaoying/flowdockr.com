# Canonical Scenario Entry Policy

- Public marketing surfaces should send first-touch traffic into the canonical scenario system first.
- Preferred entry targets are:
  - `/scenario`
  - primary canonical scenario pages such as payment, pricing, and scope attack pages
  - tool-intent-matched scenario entries when a tool page needs nearby scenario context
- `/pricing` can remain live for pricing and package details, but it is no longer the default public front door for client-situation discovery.
- Homepage, header, footer, and example/teaser modules should avoid introducing a parallel legacy `/pricing/*` situation path.
- If a new public surface recommends scenarios, default to canonical `/scenario/[slug]` entries unless the surface exists specifically to explain packages or checkout.
